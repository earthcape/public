Images need to be stored on external storage (local network, public hosting, images repositories, etc.).

Two files need to be created with the following sets of fields:

1 Attachment:

  • OriginalFileName (must be unique)
  • Url (must be accessible to your users)

2 UnitAttachment: link attachments to Units:

  • Unit (UnitID of the Unit record to link to)
  • File (file name, same as in first file)

Use built in Importer to import files in that order. Notes:

  • For this to work - your file names stored must be unique.
  • Remember when testing to save the import record after it is done to commit to database.
  • Thumbnail of the image is stored in the database and linked to as Unit.Image field that can be exposed both in list and detail views.
